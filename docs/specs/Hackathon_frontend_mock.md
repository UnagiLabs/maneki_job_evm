# フロントエンド・モック Spec（Hackathon向け）

## 0. ゴール

* 目的：JPYC（EVM）× World ID の「スキル売買」サービスの雰囲気を、**実データ無し**で体験できる “見た目・導線・動き” 重視のモックを作る
* 範囲：フロントのみ（API/DB/コントラクト接続なし）
* ただし、将来BFF/API実装に差し替えやすいように「データ取得層」を分離しておく

---

## 1. 想定技術（フロントのみ）

* Next.js（App Router）
* Tailwind CSS
* UI：shadcn/ui（任意）
* データ：`src/mocks/*.ts` の静的データ（JSON相当）
* ルーティング：`app/` 配下
* 画像：`public/mock/*`（ダミー画像 or gradient）

---

## 2. ルーティング（最低限）

* `/`：トップ（ココナラ風のメインページ）
* `/categories/[slug]`：カテゴリ一覧（カードグリッド＋フィルタの見た目だけ）
* `/services/[id]`：サービス詳細（価格・出品者・評価・購入ボタンの見た目）
* `/sellers/[id]`：出品者プロフィール（評価・実績・出品一覧）
* `/auth`：ログイン（World ID風UIのみ）
* `/me`：マイページ（ウォレット紐付け/プロフィール編集の見た目）
* `/orders`：取引一覧（ダミー）
* `/orders/[id]`：取引詳細（チャット・納品・承諾・差し戻し・紛争UIの見た目）

※ハッカソンの「最初の印象」重視なら、まず `/` と `/services/[id]` と `/auth` を作るのが効果大。

---

## 3. 画面仕様：トップページ（/）

ココナラのトップページに近いセクション構成で作る。 ([ココナラ][1])

### 3.1 ヘッダー（グローバルナビ）

* 左：ロゴ（クリックで `/`）
* 中央：検索ボックス（プレースホルダ「サービスを探す」）
* 右：リンク群

  * 「サービスを探す」「プロ人材を探す」「仕事を探す」「ブログを探す」相当のタブ/メニュー ([ココナラ][1])
  * 「ログイン」「会員登録」ボタン ([ココナラ][1])
* 追加（このプロダクト要素）

  * 右上に「JPYC決済」「World ID対応」バッジ（小さめ）

挙動（モック）

* 検索入力：入力すると下にサジェスト風のドロップダウン（モックデータから上位5件）
* ログイン：`/auth` に遷移
* 会員登録：`/auth?mode=signup` に遷移

### 3.2 ヒーロー（ファーストビュー）

* キャッチ：例「プロが集まるスキルマーケット」（雰囲気合わせ） ([ココナラ][1])
* サブ：例「World IDで1人1アカウント / JPYCで安心決済」
* CTA：

  * Primary「まずは無料で会員登録」相当 ([ココナラ][1])
  * Secondary「サービスを探す」
* 右側：イメージ（イラスト/モック端末）

挙動

* スクロールでヒーローが軽くフェード（Framer Motion任意）
* CTAクリックで `/auth` または `/services` へ

### 3.3 カテゴリから探す（重要）

* タイトル：「カテゴリから探す」 ([ココナラ][1])
* 表示：20個前後のカテゴリをカード（アイコン＋名前）で並べる（横スクロール or グリッド） ([ココナラ][1])
* 例カテゴリ（雰囲気）：デザイン、Web制作、動画、マーケ、ビジネス代行、IT、翻訳、士業、AI、占い、悩み相談、学習… ([ココナラ][1])

挙動

* hoverで浮く
* クリックで `/categories/[slug]`

### 3.4 人気のご利用シーン（重要）

* タイトル：「人気のご利用シーン」 ([ココナラ][1])
* 表示：8個程度をカード（画像＋タイトル）

  * 商品・サービスPR / お困りごと解決 / 独立開業 / EC運営 / メディア運営 / 店舗運営 / YouTuber / 結婚式… ([ココナラ][1])

挙動

* クリックで「絞り込みが適用されたカテゴリ一覧」っぽい画面に遷移（実装はURLクエリだけでOK）

### 3.5 会員登録CTA（中間）

* 「まずは無料で会員登録！」＋ボタン ([ココナラ][1])
* モック特典：例「取引手数料10%固定」「World IDで不正抑止」など

### 3.6 メリット（3カラム）

* タイトル：「（サービス名）のメリット」
* 3項目（ココナラの構成を踏襲）

  * 比較してお得 / オンラインで完結 / 安心・安全 ([ココナラ][1])
* ここでこのプロダクト要素を混ぜる

  * 安心・安全：World ID、エスクロー、オンチェ完結（運営が資金を動かせない）を“短文＋アイコン”で表現

### 3.7 おすすめサービス（カードグリッド）

* タイトル：「おすすめサービス」 ([ココナラ][1])
* 表示：12件程度
* カード要素

  * サムネ、タイトル、サブ説明、★評価、レビュー数、出品者名、価格（JPYC表記にする）
* 「もっと見る」リンク ([ココナラ][1])

挙動

* カードクリックで `/services/[id]`

### 3.8 おすすめユーザー（出品者）

* タイトル：「おすすめユーザー」 ([ココナラ][1])
* 表示：横スクロール
* 要素：アイコン、表示名、職種バッジ（例：デザイナー/エンジニア）、★、実績
* クリックで `/sellers/[id]`

### 3.9 フッター

* リンク群（ガイド/利用規約/特商法/問い合わせ）
* チェーン表記（MVP: Polygon想定 などは小さく）

## 4. サービス詳細ページ（/services/[id]）最低仕様

* 左：サムネ + ギャラリー
* 中央：タイトル、説明、購入前の注意、FAQ（アコーディオン）
* 右：購入カード（固定）

  * 価格（例：12,000 JPYC）
  * 納期目安
  * 購入ボタン（押すとモーダルで「JPYCで支払う（モック）」→成功トースト）
* 下部

  * 出品者情報（プロフィールカードへリンク）
  * レビュー一覧（ダミー）
  * 関連サービス（ダミー）

---

## 5. 認証モック（/auth）

* World ID風のUI

  * 「World IDで続行」ボタン
  * クリックするとモーダルで「Proof取得中…」→「検証OK」→ `/me` 遷移（擬似ログイン）
* 実装は `localStorage` で `mockSession=true` を保存するだけ

---

## 6. データ（モック）仕様

`apps/web/src/mocks/` に静的データを置く（将来API差し替えが簡単）

### 6.1 データファイル

* `mocks/categories.ts`
* `mocks/scenes.ts`
* `mocks/services.ts`
* `mocks/sellers.ts`
* `mocks/reviews.ts`

### 6.2 最低スキーマ（TypeScript）

* Category

  * `id, slug, name, icon`
* Scene

  * `id, slug, name, image`
* Service

  * `id, title, summary, priceJpyc, categorySlug, sellerId, ratingAvg, ratingCount, thumbnail`
* Seller

  * `id, name, title, avatar, ratingAvg, ratingCount, completedCount, badges[]`
* Review

  * `id, serviceId, rating, comment, authorName, createdAt`

## 7. “リッチ”に見せるための演出要件

* Skeleton表示（初回ロードっぽく見せる）

* hover/transition（カードがふわっと浮く）
* トースト通知

  * 「ログインしました」「購入フローを開始しました（モック）」など
* モーダル

  * ログイン、購入、検索サジェスト詳細
* レスポンシブ

  * ヘッダー：モバイルはハンバーガー＋検索アイコン
  * セクション：横スクロール＋スナップ

## 9. 受け入れ条件（モックとしての完成条件）

* トップページに以下のセクションが揃っている

  * ヘッダー（検索・ログイン/登録） ([ココナラ][1])
  * ヒーロー＋CTA ([ココナラ][1])
  * カテゴリから探す ([ココナラ][1])
  * 人気のご利用シーン ([ココナラ][1])
  * メリット（3カラム） ([ココナラ][1])
  * おすすめサービス ([ココナラ][1])
  * おすすめユーザー ([ココナラ][1])
* `/auth` のWorld IDモックが動き、疑似ログインできる
* サービスカード→詳細ページに遷移できる
* 価格表記がJPYCになっている（例：`12,000 JPYC`）
* 主要コンポーネントが再利用され、ページが肥大化していない
